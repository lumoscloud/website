2-Day Cisco ACI Test Drive

Offering Formats

This course is offered in the following formats:

  • Live, Instructor-Led Training (schedule)

Course Overview

In this 2-day instructor-led Nexus 9000 ACI Test Drive, you will become experienced with the planning, designing, and implementation of Cisco® Application Centric Infrastructure.

It is designed for customers, systems engineers, and field engineers who are implementing and managing Cisco ACI. The course covers the key concepts behind Cisco’s application-centric infrastructure and policy-based view of the solution.  You will learn how to bring up an ACI fabric, configure tenants, application network profiles (ANP), contracts, endpoints, VMM integration and how to connect the ACI Fabric to an external L2 and L3 networks, along with L4-L7 services. Automation of APIC REST API through scripting (JSON/XML) will be discussed.

The course will include intense hands-on labs on real physical hardware of all of the key functions of ACI. You will not share with other students during the labs. You will be provided with your own ACI Tenant, Vcenter Server, ESXi Hosts and numerous virtual machines.

Upon completion of the course, students will be familiar with:

  • Pre-Deployment Interview/Information gathering process
  • Out of the box setup and configuration of an ACI fabric
  • Scenario driven configuration principals for:
    • Multi-Tenancy/Tenant Configurations
    • Application Profiles
    • Endpoint Groups
    • Security - Built-in Stateful Firewall, Contracts
    • Virtual Machine Manager (VMM) Integration
    • Connecting the ACI fabric to external L2 and L3 networks
    • Connecting fabric to L4 – L7 services (ASA, F5, etc….)

Course Specifics

  • This course has a high degree of labs (40:60 ratio lab:lecture), and is intended to engage and challenge the student into performing the most common but also complex scenarios when migrating to Cisco Application Centric Infrastructure (ACI) Fabric
  • Hands on lab environments. Each student will have:
    • Dedicated ACI tenant with 4 ESXi hosts and 9 VM’s
    • Dedicated vCenter Appliance
    • Dedicated L2/L3 outside connections
  • Class size will be restricted to 16 students to facilitate more hands on lab time.
  • Each student will have a dedicated Tenant and dedicated ports both on the ACI Leafs and on the external Nexus 5K/6Ks, dedicated 4 ESXi servers, vCenter server and VMs.
  • Students do not share or “partner-up” to perform tasks. Each student independently performs all tasks.

Who Should Participate?

  • Solution Architects
  • System Integrators
  • End-User deploying/implementing Nexus Access Switches
  • Network Administrators
  • System Engineers

Recommended Prerequisites

  • Basic understanding of networking
  • Basic knowledge of VMware virtualization technology would be beneficial

Course Agenda

Day 1
Module 1 ACI Overview
Evolution of the modern Data Center. From the traditional 3-Tier architecture to the Spine-Leaf Architecture. Why the industry moving to VxLAN and to the Policy driven Data Center.
Module 2 ACI Fabric Switches
Nexus 9000 platforms in stand-alone and ACI mode. Understanding hardware and software capabilities of the Nexus 9000 platform
Module 3 Application Policy Infrastructure Controller
How the Cisco APIC provides fabric configuration and Application Network Policy creation and deployment. in addition to providing resiliency, redundancy, scalability and monitoring
Lab 1 ACI Fabric Discovery
Configure ACI from initial start up through to simple configurations. Get familiar with the ACI GUI
Module 4 Fabric Access and Resource Pools
How to create and use ACI policies and resource pools. These objects are used to building a scalable switch fabric
Lab 2 Fabric Access Policy Configuration
Create host access policies and pools for the ACI fabric. Switch, Interface and Policy groups will be defined, along with VLAN Pools
Module 5 Tenant Logical Model
Understanding the object model of building an application network profile, such as End Point Group (EPG), Provider and Consumer constructs
Lab 3 Tenant and Application Polices
Create the logical tenant space. Continue on by creating the required basic networking hierarchy, VRF, Bridge Domain, Subnets, Gateways. Create the initial Application Network Profile and EPG definitions
Module 6 Fabric Operation and Forwarding
How does the fabric operate. Go under the covers and see how packets are learned and forwarded inside the fabric. How ACI can connect to bare metal servers or hypervisors running VLAN, VxLAN, NVGRE and forward packets seamlessly between any of these protocols
Day 2
Module 7 Virtual Machine Management
Integration of VMware, Microsoft and OpenStack into the ACI environment will be discussed. Comprehension of how the flow and operations of each hypervisor solution is implemented in ACI.
Lab 4 Configuring VMM Integration
In this lab, we will configure ACI integration with VMware ESXi hosts and provide EPG configuration as port groups in the vCenter environment.
Module 8 Communication Using Contracts (ACLs)
Discuss how contracts are used to both allow EPG communication and also create route leaking between VRFs. Relationship to ACL and TCAM security polices. How contracts can be used in a Network Centric design or an Application Centric Design
Lab 5 Create and Apply Contracts
Create Contracts, filters, subjects. Apply the contracts between EPGs using the Provide and Consume methodology to allow data to flow between EPGs
Module 8 Layer 3 External
Describing how to connect a layer 3 network outside the ACI fabric to be integrated with ACI. Understand connectivity requirements to external routers and the use of Route Reflectors in the ACI Fabric
Lab 6 Establishing Layer 3 External Connectivity
In this lab, the student will configure the ACI fabric to connect to an external L3 network. Configuring vPC connectivity to external L3 switches. Creation of external networks. Advertising internal subnets from the ACI fabric
Module 9 Layer 2 External
Describing how to connect a layer 2 network outside the ACI fabric to be integrated with ACI.
Lab 7 Layer 2 Connectivity
Attach to legacy L2 network. Extend the ACI EPG to the legacy L2 network. Use contracts to allow data to flow between the ACI and Legacy networks
Module 10 Integrating L4-7 with ACI
Understanding how to integrate layer 4 – 7 services like firewall and load balancers. Discussions of supported vendors and implementation of the services to the EPGs.
Lab 8 L4-L7 Service Insertion
Insert a L4-L7 device between EPGs using the ACI service graph
Module 11 Operating & Troubleshooting ACI
Discuss integration to existing networks, integration and migration scenarios. Understand how to manage and troubleshoot an ACI fabric after deploying the environment. Walk through of tools like API inspector and Visore
